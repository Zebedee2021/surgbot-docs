# 系统安全约束

!!! danger "安全红线"
    安全约束是系统硬性边界，任何模块的设计和实现不得违反以下约束。约束变更须经全员评审。

## 全局安全指标

| 指标名称 | 目标值 | 当前值 | 状态 | 验证方式 |
|---------|--------|-------|------|---------|
| 急停响应时间 | ≤80ms | 待测量 | ⬚ | 硬件计时测试 |
| 碰撞率 | 0（目标） | 待测量 | ⬚ | 现场测试统计 |
| 夹取成功率 | ≥97% | 待测量 | ⬚ | 批量夹取测试（≥100 次） |
| 递交成功率 | ≥98% | 待测量 | ⬚ | 批量递交测试（≥100 次） |
| 端到端延迟 | ≤2.5s | 待测量 | ⬚ | 全链路计时（语音→递交完成） |

> 状态标记：✅ 已达标 / ⏳ 测试中 / ⬚ 待测量

## 模块安全职责矩阵

| 模块 | 安全职责 | 约束指标 | 验证方式 |
|------|---------|---------|---------|
| 感知模块 | 误识别防护：过滤低置信度目标 | 检测置信度阈值 ≥0.8 | 对比测试（正常/异常样本） |
| NLP 模块 | 指令误解防护：拒绝无法匹配的器械名称 | 器械名称识别 F1 ≥0.95 | 标准测试集评测 |
| 决策模块 | 动作安全校验：目标位姿合理性检查 | 动作空间边界 + 碰撞预检 | 仿真回归测试 |
| 执行模块 | 物理安全保障：急停、力控、碰撞检测 | 急停 ≤80ms + 力控超限保护 | 硬件测试 |
| 仿真模块 | Sim2Real gap 控制 | 位姿迁移误差 ≤2cm | 仿真-真实对比实验 |

## 安全工作流

!!! warning "执行前三重确认"
    每次执行动作指令前，系统须依次完成以下校验，任一失败则中止执行：

1. **目标位姿验证** — 检查目标坐标是否在机械臂工作空间内
2. **路径安全检查** — 检查运动路径是否与已知障碍物冲突
3. **力传感器预读** — 确认力传感器读数正常，排除夹爪异常

详见：[安全策略](../modules/execution/safety_strategy.md)

## 异常场景与响应

| 异常场景 | 触发条件 | 响应动作 | 恢复方式 | 责任模块 |
|---------|---------|---------|---------|---------|
| 急停触发 | 物理按钮 / 语音"停" / 软件指令 | 立即停止所有关节运动 | 手动复位后重启 | 执行 |
| 碰撞检测 | 力传感器超过阈值 | 停止运动 + 回退 5mm | 自动回退后等待指令 | 执行 |
| 识别失败 | 置信度 < 0.8 或无目标 | 拒绝执行，语音提示 | 等待下一条指令 | 感知 |
| 指令无法解析 | NLP 输出无法匹配词库 | 拒绝执行，语音提示重说 | 等待下一条指令 | NLP |
| 路径异常 | 运动规划无可行路径 | 中止执行，报告异常 | 人工干预调整位置 | 决策+执行 |
| 力控超限 | 夹持力/递交力超过安全值 | 松开夹爪 + 停止运动 | 自动松开后等待指令 | 执行 |
| 通信中断 | SDK 连接超时（>500ms） | 停止所有指令下发 | 自动重连，3 次失败后报警 | 执行 |
| 模型推理超时 | 决策模型响应 >3s | 跳过本次，保持当前状态 | 等待下一条指令 | 决策 |
