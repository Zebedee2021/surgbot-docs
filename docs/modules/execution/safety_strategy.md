# 安全策略

## 安全需求

手术室场景对安全性要求极高，机械臂的任何异常动作都可能造成严重后果。

## 安全机制设计

### 急停机制

- 物理急停按钮
- 语音指令"暂停机器人"
- 软件层面的紧急停止接口

### 碰撞检测

- 基于力传感器的碰撞检测
- 运动前的路径安全检查
- 工作空间边界限制
- CR5AF 内置碰撞检测信号，可通过 SDK `get_collision_status()` 实时查询
- 检测到碰撞后自动触发安全回退，回到预设安全姿态
- 支持碰撞灵敏度分级设置（高/中/低），手术场景建议使用「高灵敏度」

### 力反馈

- 递交时通过力反馈确认医生已握住器械
- 夹取时检测是否稳定夹持

### 异常处理

!!! danger "必须解决"
    目前存在目标位置识别错误时机械臂仍然运动的安全隐患。需要在执行前综合验证目标位置的合理性，路径规划时加入安全检查。

### 执行前安全校验

每次运动执行前，系统需要完成三重确认：

1. **目标位姿合理性验证** — 检查目标坐标是否在机械臂工作空间内，是否与已知障碍物冲突
2. **路径安全检查** — 验证中间路径点是否穿越禁区（如医护人员活动区域）
3. **力传感器预读** — 在启动运动前读取力传感器基准值，确保夹爪和臂体状态正常

!!! info "设计依据"
    此校验机制基于核心开发框架中「安全保障层」的设计要求，与状态机中 Execute 状态的入口检查对应。详见 [系统架构 - 状态机设计](../overview/architecture.md#状态机设计)。

## 安全等级

| 场景 | 响应 | 要求 |
|------|------|------|
| 急停触发 | 立即停止所有运动 | 响应时间 <= 80ms |
| 碰撞检测 | 停止并回退到安全位 | - |
| 识别异常 | 暂停执行，等待确认 | - |
| 通信中断 | 自动进入安全模式 | - |
| 路径异常 | 重新规划或停止执行 | 执行前校验不通过时触发 |
| 力控超限 | 松开夹爪并回退 | CR5AF 力传感器阈值超限时触发 |
